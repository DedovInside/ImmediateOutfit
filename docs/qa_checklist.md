# ImmediateOutfit — Live QA Checklist

Чек-лист ручного прохождения бота и dashboard перед демонстрацией и полевыми тестами.
В каждом пункте отметь: ✅ работает / ⚠️ нюанс / ❌ баг.

## Подготовка
- [ ] Активирован `.venv312`: `source .venv312/bin/activate`
- [ ] `BOT_TOKEN` задан в `.env`
- [ ] Для AI-сценариев опционально заданы `AI_ENABLED=true` и `DEEPSEEK_API_KEY`; без них должен работать rule-based fallback.
- [ ] Бот запущен: `python bot.py` → в логах `🚀 Бот OutfitNow запущен!`
- [ ] Dashboard запущен параллельно: `uvicorn main:app --reload`

---

## Сценарий 1. Основной подбор (`/start` → «Подобрать образ»)
- [ ] `/start` показывает приветствие и 6 кнопок главного меню.
- [ ] «Подобрать образ» открывает шаг «Для кого» (при пустом профиле) или сразу шаг «Куда идёшь» (если профиль уже есть).
- [ ] Все 8 шагов проходят без ошибок: gender → occasion → weather → activity → priority → mood → budget → style.
- [ ] На шаге weather кнопка «📍 По моему городу» отображается. Клик → запрос города → ввод «Москва» → бот отвечает температурой и продолжает к activity. В `/stats` события `weather_auto_attempted` и `weather_auto_succeeded`.
- [ ] При вводе несуществующего города (например, «xyzxyzxyz») — fallback на ручной выбор + событие `weather_auto_failed`.
- [ ] `/skip` на шаге `weather_city` возвращает на ручной выбор погоды.
- [ ] После последнего шага показывается **2 образа**.
- [ ] Каждая карточка содержит: items, описание, tip, причины (`why_it_fits`), а для командных подборок — палитру, советы по носке и артикулы.
- [ ] Кнопки `Сохранить`, `Почему подходит`, `Референс`, `Где искать` доступны и кликабельны, если для образа есть референс/артикулы.
- [ ] `Показать ещё` подгружает следующую пару образов, **не повторяя** уже показанные.
- [ ] `Оценить ⭐ 1/3/5` пишет событие `result_feedback` (проверить в `/stats`).
- [ ] После 👎 или 🤔 бот спрашивает «что не зашло?». Ввод текста → `result_feedback_comment` в `/stats`. `/skip` или кнопка «Пропустить» закрывают шаг без события.
- [ ] После 👍 бот **не** спрашивает комментарий.

## Сценарий 2. Быстрый подбор
- [ ] Запускается только если профиль уже заполнен (есть gender + budget или style).
- [ ] **Пропускает** шаги gender / budget / style, оставляет 3-4 шага.
- [ ] Завершается выдачей 2 образов как в основном сценарии.

## Сценарий 3. «Под мою вещь»
- [ ] Кнопка открывает запрос на ввод текста («Напиши вещь…»).
- [ ] При включённом DeepSeek ввод «черные широкие джинсы, дождь, casual» сразу возвращает 2-3 AI-варианта без дополнительной анкеты.
- [ ] При успешном ответе DeepSeek пишется событие `ai_item_outfits_generated`.
- [ ] При выключенном DeepSeek или ошибке API сценарий падает обратно на кнопочную анкету и пишет `ai_item_outfits_fallback`.
- [ ] Ответ не содержит выдуманных артикулов: только вещи, цвета, советы и ограничения.

## Сценарий 4. «Проверить мой образ»
- [ ] Открывается запрос на ввод описания текущего лука.
- [ ] Ввод «джинсы и кроссовки» возвращает структурированный разбор: headline, score_label, strengths, concerns, suggestions.
- [ ] При включённом DeepSeek событие `review_completed` содержит `source=deepseek`; при ошибке/выключении — `source=rule_based`.
- [ ] Короткий мусорный ввод вроде «Хы» не отправляется в AI и просит описать образ подробнее.
- [ ] Кнопки `Оценить ⭐ 1/3/5` пишут событие `review_feedback`. После 👎/🤔 бот тоже спрашивает текстовый комментарий — пишет в `review_feedback_comment`.

## Сценарий 5. «Мой профиль»
- [ ] Открывает список секций: цвета, anti-preferences, key items.
- [ ] После ввода `чёрный, бежевый` → `/stats` показывает рост `profile_completion_rate`.
- [ ] При повторном «Подобрать образ» — образы с этими цветами должны подниматься в выдаче.
- [ ] Anti-preferences (например, «карго») — образы с карго **не должны** быть в топе.

## Сценарий 6. «Мои сохранённые»
- [ ] Сразу после сохранения карточки кнопкой `Сохранить` — список не пустой.
- [ ] Повторное нажатие `Сохранить` на ту же карточку **не удваивает** запись (проверка `INSERT OR IGNORE`).

## Сценарий 7. Premium
- [ ] Открывает заглушку с призывом «Хочу узнать».
- [ ] Клик `Хочу узнать` инкрементит `premium_interest` в `/stats` (видно в карточке dashboard).

## Dashboard (http://127.0.0.1:8000)
- [ ] `/` рендерит HTML без 500 и без видимых JS-ошибок.
- [ ] На главной странице блок «Голоса и комментарии» показывает распределение 👍/🤔/👎 и список последних текстовых комментариев (после ввода в боте).
- [ ] `/stats` отдаёт JSON с заполненными секциями `users`, `funnel`, `engagement`, `quality`, `events`; `quality.result_feedback_distribution` отражает счётчики.
- [ ] `/outfits` показывает базовые и командные образы; `/outfits?gender=male` фильтрует.
- [ ] `/artifacts/metrics`, `/artifacts/sprint_map`, `/artifacts/roadmap`, `/artifacts/unit_economics` отдают 200.
- [ ] `/health` отдаёт `{"status":"ok", ...}`.
- [ ] В логах `uvicorn` нет `DeprecationWarning` про `on_event`.

- Что ожидал увидеть.
- Что увидел реально (текст ошибки или поведение).

## Что НЕ проверяется в этом чек-листе (вне скоупа)
- Photo-review / AI try-on (вне ближайшего контура).
- Полная фотобаза гардероба пользователя (в MVP вместо неё профиль, key items и подбор под вещь).
- Реальная оплата Premium (в MVP собирается `premium_interest`; Telegram Stars — следующий этап).
